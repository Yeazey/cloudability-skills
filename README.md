# Cloudability Kiro Skills + Dashboards

Single repo for all IBM Cloudability FinOps tooling — Kiro CLI skills and the unified Python dashboard generator.

## Repository Structure

```
├── cldy/                    # MCP context skill (load at session start)
├── execdash/                # Executive dashboard skill
├── archdash/                # Multi-cloud architecture dashboard skill
├── containersdash/          # Kubernetes container cost dashboard skill
├── dailycheckin/            # Daily FinOps standup skill
└── dashboards/              # Unified Python dashboard generator (all dashboards)
    ├── pyproject.toml
    └── src/cloudability_dashboards/
        ├── client.py        # Direct Cloudability V3 API client (httpx)
        ├── cli.py           # cldy-dash CLI entry point
        ├── generators/      # One generator per dashboard
        └── templates/       # Jinja2 templates (shared base + per-dashboard)
```

## Skills

| Skill | Trigger | What it does |
|-------|---------|-------------|
| `cldy` | `/cldy`, session start | Loads full Cloudability MCP context — environments, auth, tools, workflows |
| `execdash` | exec dashboard, cloud cost summary | Generates executive FinOps dashboard |
| `archdash` | multi-cloud, architecture, chip breakdown | Generates multi-cloud architecture dashboard |
| `containersdash` | container dashboard, k8s costs, idle resources | Generates K8s container cost dashboard |
| `dailycheckin` | daily checkin, finops standup | Runs FinOps daily standup (markdown report) |

### Installation

Copy skill folders into `~/.kiro/skills/`:

```bash
git clone https://github.com/Yeazey/cloudability-kiro-skills.git
cp -r cloudability-kiro-skills/{cldy,execdash,archdash,containersdash,dailycheckin} ~/.kiro/skills/
```

## Dashboard Generator

All dashboards are generated by a single Python project in `dashboards/`.

### Setup

```bash
cd dashboards
uv sync

# Set credentials (or create dashboards/.env)
export CLOUDABILITY_OPEN_TOKEN="your-token"
export CLOUDABILITY_ENVIRONMENT_ID="your-env-id"
```

### Usage

```bash
cd dashboards
uv run cldy-dash executive     # Executive cloud cost dashboard
uv run cldy-dash multicloud    # Multi-cloud architecture breakdown
uv run cldy-dash containers    # Kubernetes container cost dashboard
uv run cldy-dash checkin       # Daily FinOps standup (markdown to stdout)
uv run cldy-dash checkin --json  # JSON output for programmatic use
```

### Design Principles

- **One language**: Python everywhere. No Node.js, no subprocess spawning.
- **Direct API calls**: httpx calls Cloudability V3 API directly. No MCP server dependency.
- **Shared templates**: Jinja2 base template provides consistent dark theme across all dashboards.
- **Minimal dependencies**: `httpx` + `jinja2`. That's it.
- **Self-contained output**: Each generated HTML file is standalone with embedded data + Chart.js CDN.

### Dependencies

```toml
[project]
dependencies = [
    "httpx>=0.27.0",
    "jinja2>=3.1.0",
]
```
